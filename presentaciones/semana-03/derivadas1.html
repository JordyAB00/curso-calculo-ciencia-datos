¿Cómo deriva una computadora?

Tres caminos, una misma matemática

El problema

Ustedes pueden derivar a mano una función como f(x) = sin(x)·e^(-x²) con paciencia. Pero un modelo de lenguaje como GPT-5 tiene cerca de 1,8 billones de parámetros. Derivar a mano respecto a cada uno tomaría más años que la edad del universo.

Camino 1: numérico

Aproximamos con diferencias finitas. Es rápido pero impreciso. Lo usaron desde Newton.

Camino 2: simbólico

La computadora aplica las reglas algebraicas como ustedes. Es exacto pero lento para funciones complejas.

Camino 3: automático

Construye un grafo de operaciones y aplica la regla de la cadena. Es el corazón del deep learning.

Hoy van a ver los tres caminos sobre la misma función, lado a lado.

Camino 1: derivada numérica con NumPy

Aproximación por diferencias finitas (la idea del cociente diferencial)

import numpy as np import matplotlib.pyplot as plt # Función: f(x) = x³ - 3x x = np.linspace(-3, 3, 500) f = x**3 - 3*x # Derivada con diferencias centradas dfdx = np.gradient(f, x) # Visualizar lado a lado fig, ax = plt.subplots(1, 2) ax[0].plot(x, f) ax[1].plot(x, dfdx) plt.show()

Diferencias centradas

np.gradient usa (f(x+h) - f(x-h)) / (2h), con error O(h²): más preciso que la fórmula básica O(h).

Lectura de la gráfica

Donde la derivada cruza cero (x = ±1), la función original tiene un máximo o un mínimo local. Pura geometría aplicada.

Lo nuevo: scipy.differentiate

SciPy 1.15 (enero 2025) jubiló el viejo scipy.misc.derivative

Hasta hace poco, la función de SciPy para derivar numéricamente estaba en scipy.misc.derivative, pero fue eliminada en SciPy 1.12 y reemplazada por un módulo nuevo y vectorizado con detección de error automática.

from scipy.differentiate import derivative # Función analítica conocida def f(x): return x**3 - 3*x # Derivada en varios puntos res = derivative(f, 2.0) print(f"f'(2) = {res.df:.10f}") print(f"error estimado: {res.error:.2e}") print(f"iteraciones: {res.nit}") print(f"éxito: {res.success}")

Camino 2: derivada simbólica con SymPy

La computadora aplica las reglas como ustedes — exacto, sin errores de redondeo

from sympy import symbols, diff, sin, exp, simplify x = symbols('x', real=True) # Misma función, derivada exacta f = x**3 - 3*x df = diff(f, x) print("df/dx =", df) # Una más interesante g = sin(x) * exp(-x**2) dg = simplify(diff(g, x)) print("dg/dx =", dg) # Evaluar en un punto print("df/dx en x=2 =", df.subs(x, 2))
python — sympy
>>> print("df/dx =", df) df/dx = 3*x**2 - 3 >>> print("dg/dx =", dg) dg/dx = (-2*x*sin(x) + cos(x))*exp(-x**2) >>> print("df/dx en x=2 =", df.subs(x, 2)) df/dx en x=2 = 9 # Resultado exacto: no es 9.0000000001, # es 9. Entero. Matemática pura.

Ventaja pedagógica

SymPy es su "juez imparcial": cuando deriven a mano, lo verificarán aquí. Cuando duden de una integral en la semana 9, lo confirmarán aquí. Lo van a usar en cada taller del curso.

Camino 3: diferenciación automática

Ni numérica ni simbólica: algo más inteligente

La idea

El autodiff no aproxima ni resuelve simbólicamente. Construye un grafo computacional con cada operación elemental (suma, multiplicación, sin, exp) y aplica la regla de la cadena hacia atrás. Resultado: derivada exacta hasta el último dígito de precisión flotante, con costo similar al de evaluar la función original.

Modo forward

Propaga (valor, derivada) hacia adelante. Eficiente cuando hay pocas variables y muchas salidas.

Modo reverse

Construye el grafo y aplica la regla de la cadena hacia atrás. Eficiente cuando hay muchas variables y pocas salidas (caso típico en IA).

Estándar de industria

PyTorch, JAX, TensorFlow, MLX, Mojo: todos usan autodiff en modo reverse para entrenar modelos.

"Backpropagation no es más que diferenciación automática en modo reverse aplicada al cálculo de gradientes de una función de pérdida." Zhang, Lipton, Li y Smola — Dive into Deep Learning (2023)

Demo en vivo: torch.autograd

Calculemos f'(2) para f(x) = x³ - 3x con cuatro líneas

import torch # 1. Crear el tensor con seguimiento de gradiente x = torch.tensor(2.0, requires_grad=True) # 2. Construir la función (grafo en silencio) f = x**3 - 3*x # 3. Recorrer el grafo hacia atrás f.backward() # 4. Leer el resultado print(f"f(2) = {f.item()}") print(f"f'(2) = {x.grad.item()}")
python — torch.autograd
>>> x = torch.tensor(2.0, requires_grad=True) >>> f = x**3 - 3*x >>> f.backward() >>> print(f"f(2) = {f.item()}") f(2) = 2.0 >>> print(f"f'(2) = {x.grad.item()}") f'(2) = 9.0 ✓ Coincide con el cálculo manual y con SymPy: 3(2²) - 3 = 9

Esto mismo, con un grafo de millones de nodos en lugar de tres, es lo que entrena a ChatGPT, a Gemini y a Claude. Es exactamente la misma técnica, solo que más grande.

ReLU: la función no derivable en cero

La estrella del deep learning, con un detalle incómodo

python — ¿qué hace PyTorch en x=0?
>>> for p in [-1.0, 0.0, 1.0]: ... x = torch.tensor(p, requires_grad=True) ... y = torch.relu(x) ... y.backward() ... print(f"x={p:+.1f}: ReLU={y.item()}, " ... f"derivada={x.grad.item()}") x=-1.0: ReLU=0.0, derivada=0.0 x=+0.0: ReLU=0.0, derivada=0.0 ← x=+1.0: ReLU=1.0, derivada=1.0 # En x=0, PyTorch escoge 0 como # subgradiente válido (podría ser # cualquier valor entre 0 y 1).

La respuesta: subgradientes

Un subgradiente es cualquier número entre las dos pendientes laterales. Para ReLU en 0, cualquier valor entre 0 y 1 es válido. PyTorch toma 0 por convención; TensorFlow también; JAX toma 0. Funciona porque la probabilidad de caer exactamente en x = 0 sobre datos continuos es esencialmente cero.

La función de Weierstrass y por qué importa

El monstruo de 1872 que rompió la intuición geométrica

El "monstruo"

En 1872, Karl Weierstrass construyó:

W(x) = Σ aⁿ cos(bⁿπx)

con 0 < a < 1 y b entero impar tal que ab > 1 + 3π/2. Es continua en todo punto y derivable en ninguno.

Reacciones de la época

"Una lamentable calamidad." Charles Hermite, 1893
"Un atentado contra el sentido común." Henri Poincaré, 1899

¿Por qué importa hoy? Porque las redes neuronales con ReLU producen funciones que se comportan como Weierstrass: continuas casi en todas partes, pero con esquinas en cantidades enormes. La matemática que justifica que PyTorch funcione sobre ellas se llama "diferenciación PAP" (Lee et al., NeurIPS 2020).

La cinemática moderna: del cohete al gradiente

La misma idea, 350 años después

Newton, 1687

Problema: describir el movimiento de un planeta.

Solución: tasa de cambio instantánea de la posición.

v(t) = s'(t)
a(t) = s''(t)

OpenAI, 2026

Problema: entrenar un modelo de lenguaje.

Solución: tasa de cambio instantánea del error.

∇θ L(θ) = derivada del loss
θ ← θ - η · ∇θ L

El gradiente del loss respecto a los parámetros es, literalmente, "qué tan rápido cambia el error si movemos cada parámetro un poquito". Es la cinemática de Newton, aplicada a un espacio de un billón de dimensiones en lugar de tres.

Errores comunes (la trampa del gradiente acumulado)

Lo más reportado por estudiantes de IA en su primera red

import torch # Calculamos f'(2) tres veces seguidas... x = torch.tensor(2.0, requires_grad=True) for i in range(3): f = x**3 - 3*x f.backward() print(f"Iteración {i}: x.grad = {x.grad.item()}") # ¿Qué esperan? ¿9, 9, 9? # PyTorch ACUMULA gradientes # entre llamadas a backward()
python — gradiente acumulado
Iteración 0: x.grad = 9.0 Iteración 1: x.grad = 18.0 ← se sumó Iteración 2: x.grad = 27.0 ← se sumó otra vez # Solución: reiniciar antes # de cada backward() >>> x.grad.zero_() >>> f.backward() >>> print(x.grad.item()) → 9.0 # ahora sí, cada vez

Por qué PyTorch fue diseñado así

En entrenamiento real, a veces conviene acumular gradientes a propósito (gradient accumulation) cuando el batch no cabe en la memoria de la GPU. PyTorch decidió que acumular fuera el comportamiento por defecto, así que recordar zero_() es responsabilidad del programador.

Lo que se llevan de hoy

Tres ideas para no olvidar

1. La derivada es una idea, no un cálculo

El límite del cociente diferencial es la misma idea ya sea que la apliquen a mano, con SymPy, o con torch.autograd. Las herramientas cambian; el concepto, no.

2. Cada herramienta tiene su lugar

NumPy para datos discretos, SymPy para entender la fórmula, PyTorch para entrenar modelos con millones de parámetros. Un buen científico de datos elige según el problema.

3. La frontera entre cálculo e IA es porosa

El descenso de gradiente, la backpropagation, los optimizadores Adam y AdamW son todos cálculo diferencial aplicado. Lo que vieron hoy es el motor matemático del estado del arte en inteligencia artificial.
