NumPy: vectorización y broadcasting

La razón por la que el descenso de gradiente es viable

La diferencia que cambia todo

Cuando entrenan un modelo de IA, calculan operaciones sobre millones de parámetros simultáneamente. Hacerlo con loops Python tomaría horas. NumPy lo hace en milisegundos con la misma sintaxis matemática.

import numpy as np # Mallas para cálculo x = np.linspace(-5, 5, 1000) # 1000 puntos entre -5 y 5 y = np.sin(x) * np.exp(-x**2/10) # Operación vectorizada # Broadcasting: una idea poderosa X, Y = np.meshgrid(x, x) # Grilla 2D Z = np.sin(X) * np.cos(Y) # f(x,y) en cada punto # Esto reemplaza un doble loop de 1000x1000 iteraciones

NumPy aplicado al cálculo

Derivadas e integrales numéricas en 2 líneas

import numpy as np # Definimos una función f(x) = sin(x) x = np.linspace(0, 2*np.pi, 100) f = np.sin(x) # Derivada numérica con diferencias centradas dfdx = np.gradient(f, x) # Aprox. de cos(x) # Integral numérica con la regla del trapecio integral = np.trapezoid(f, x) # Aprox. de ∫sin(x)dx en [0, 2π] = 0 print(f"Integral aproximada: {integral:.6f}") # Output: Integral aproximada: 0.000000

np.gradient

Calcula la derivada numérica usando diferencias centradas. Será la base para entender qué hace torch.autograd internamente.

np.trapezoid

Regla del trapecio compuesta. Cuando no exista primitiva analítica, esta es la forma rápida de integrar numéricamente.

Atención: NumPy moderno

APIs deprecadas que aún aparecen en tutoriales viejos

Si encuentran código con esto en internet, está desactualizado

NumPy 2.0 (agosto 2023) marcó muchas funciones como obsoletas. NumPy 2.4 ya las eliminó. Estos son los reemplazos que deben usar:

Antiguo (no usar) Moderno (usar siempre) Por qué
np.trapz(y, x) np.trapezoid(y, x) Eliminado en NumPy 2.4
np.random.seed(42) rng = np.random.default_rng(42) Patrón moderno desde NumPy 1.17
np.NaN np.nan Removido en NumPy 2.0
np.float float o np.float64 Alias confuso eliminado

Si ChatGPT o Stack Overflow les sugiere código con las APIs antiguas, está respondiendo desde su memoria vieja. Verifiquen siempre contra la documentación oficial.

Matplotlib moderno: interfaz orientada a objetos

Cambio mental clave para todo el curso

# Estilo antiguo (no usar) import matplotlib.pyplot as plt plt.plot(x, y) plt.title("Función") plt.xlabel("x") plt.show() # Problema: cuando hay varias # gráficas en una figura, # el estado global se vuelve # un nudo difícil de mantener
# Estilo moderno (recomendado) import matplotlib.pyplot as plt fig, ax = plt.subplots( figsize=(8, 5), layout='constrained' ) ax.plot(x, y, label='f(x)') ax.set_title("Función") ax.set_xlabel("x") ax.legend() plt.show()

Adoptar el patrón fig, ax = plt.subplots() desde el día uno. La documentación oficial lo recomienda explícitamente y escala mucho mejor cuando empecemos a graficar funciones, derivadas e integrales en una sola figura.

Visualizando el cálculo

Función, derivada e integral en una sola figura

import numpy as np import matplotlib.pyplot as plt x = np.linspace(-3, 3, 500) f = np.exp(-x**2) # Función original df = np.gradient(f, x) # Su derivada F = np.cumsum(f) * (x[1]-x[0]) # Su integral acumulada fig, axes = plt.subplots(1, 3, figsize=(14, 4)) axes[0].plot(x, f); axes[0].set_title("f(x) = e^(-x²)") axes[1].plot(x, df); axes[1].set_title("f'(x)") axes[2].plot(x, F); axes[2].set_title("∫f(x)dx (acumulada)") plt.show()

Lo que verán en la semana 9: superficies 3D

Cuando entremos a cálculo multivariable, este mismo patrón se extenderá a plot_surface y contour para visualizar funciones f(x,y) y sus gradientes en el espacio. Por eso vale la pena dominar la interfaz orientada a objetos ahora.

SymPy: el matemático simbólico

Cálculo exacto, no aproximaciones

from sympy import symbols, sin, cos, exp, limit, diff, integrate, oo x = symbols('x', real=True) # Cálculo de límites (semana 2) limit(sin(x)/x, x, 0) # Resultado: 1 # Derivadas (semana 3-4) diff(sin(x)**2, x) # 2·sin(x)·cos(x) # Integrales (semanas 8-10) integrate(exp(-x**2), (x, -oo, oo)) # √π (¡exacto!) # Series de Taylor (semanas 11-12) exp(x).series(x, 0, 5) # 1 + x + x²/2 + x³/6 + ...

SymPy es nuestro juez imparcial: cuando calculen una derivada a mano, lo verificarán aquí. Cuando duden de una integral, lo confirmarán aquí. La matemática no se la deben creer al profesor, se la deben demostrar ustedes mismos.

El puente: lambdify

De SymPy simbólico a NumPy numérico

El problema y la solución

SymPy calcula derivadas en formato simbólico, pero para graficarlas necesitamos evaluarlas sobre miles de puntos. lambdify convierte cualquier expresión de SymPy en una función Python que acepta arrays de NumPy.

from sympy import symbols, sin, diff, lambdify import numpy as np import matplotlib.pyplot as plt # 1. Cálculo simbólico x = symbols('x') f_sym = sin(x) * x**2 df_sym = diff(f_sym, x) # Derivada simbólica # 2. Convertir a función NumPy f_num = lambdify(x, f_sym, 'numpy') df_num = lambdify(x, df_sym, 'numpy') # 3. Evaluar y graficar xs = np.linspace(-3, 3, 500) plt.plot(xs, f_num(xs), label='f(x)') plt.plot(xs, df_num(xs), label="f'(x)")

SciPy: la caja de herramientas industrial

Vista panorámica de lo que viene

Qué es SciPy

Lo que NumPy es a un martillo, SciPy es a una caja completa de herramientas profesionales. Es la implementación de referencia de algoritmos científicos en Python.

# Integración numérica robusta (semana 10) from scipy import integrate resultado, error = integrate.quad(lambda x: x**2, 0, 4) # Output: (21.333..., 2.37e-13) — valor y estimación de error # Optimización (compararemos con descenso de gradiente) from scipy.optimize import minimize resultado = minimize(lambda x: (x-3)**2, x0=0) # Output: x = 3.0 (el mínimo de la parábola) # Transformada de Fourier (semana 11) from scipy.fft import fft espectro = fft(señal)

No tienen que aprender SciPy completo hoy. Solo necesitan saber que existe y para qué sirve. Cada vez que lleguemos a un tema nuevo, abriremos el submódulo correspondiente.

PyTorch: el motor de la IA moderna

NumPy con dos superpoderes

¿Qué es PyTorch?

Es la librería que entrenó a ChatGPT, a Stable Diffusion, a los modelos de Tesla Autopilot y a casi cualquier IA moderna que se les ocurra. Es desarrollada por Meta (Facebook) y mantenida por la Linux Foundation.

Superpoder 1: GPU

Mueve los cálculos a una tarjeta gráfica con un solo comando. Hasta 100x más rápido que CPU.

Superpoder 2: autograd

Calcula derivadas automáticamente. Le pides la derivada y PyTorch te la entrega sin que escribas la fórmula.

Compatibilidad

Los tensores de PyTorch se comportan casi idénticos a los arrays de NumPy. Lo que ya saben se transfiere.

Versión que usaremos

PyTorch 2.10 viene preinstalado en Colab. La última estable es 2.11 (marzo de 2026). Cualquiera de las dos funciona perfectamente para el curso.

Demo en vivo: una derivada en 4 líneas

Sin reglas memorizadas, sin papel, sin errores

import torch # Calculemos f'(2) para f(x) = x³ + 2x² - 5x + 1 # 1. Declarar la variable y pedir a PyTorch que rastree gradientes x = torch.tensor(2.0, requires_grad=True) # 2. Definir la función f = x**3 + 2*x**2 - 5*x + 1 # 3. Calcular la derivada automáticamente f.backward() # 4. Leer el resultado print(f"f(2) = {f.item()}") print(f"f'(2) = {x.grad.item()}") # Output: # f(2) = 7.0 # f'(2) = 15.0 (porque f'(x) = 3x² + 4x - 5, en x=2: 12+8-5=15)

Esto es lo que entrenó a ChatGPT, multiplicado por miles de millones de parámetros. La idea matemática es exactamente la misma que veremos en las semanas 4 a 6 con descenso de gradiente.
