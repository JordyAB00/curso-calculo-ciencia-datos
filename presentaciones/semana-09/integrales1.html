La antiderivada en Python: SymPy

Su juez simbólico confirma lo hecho a mano (con una trampa)

import sympy as sp x = sp.symbols('x') # Integral indefinida (antiderivada) sp.integrate(x**2 + x + 1, x) # -> x**3/3 + x**2/2 + x # OJO: SymPy NO agrega la constante C sp.integrate(x**2, x) + sp.Symbol('C') # -> C + x**3/3

El detalle que cuesta puntos

SymPy devuelve una antiderivada, sin la + C: es una convención del programa. A mano ustedes sí deben ponerla. La gráfica muestra por qué: cada C es una de esas curvas paralelas.

Suma de Riemann e integral en Python

Lo del pizarrón, ahora vectorizado y con las herramientas listas

import numpy as np from scipy.integrate import trapezoid, simpson, quad def riemann(f, a, b, n, metodo="midpoint"): dx = (b - a) / n if metodo == "left": x = np.linspace(a, b - dx, n) elif metodo == "right": x = np.linspace(a + dx, b, n) else: x = np.linspace(a + dx/2, b - dx/2, n) return np.sum(f(x) * dx) f = lambda x: x**2 for n in (4, 100, 1000): print(n, riemann(f, 0, 2, n)) # -> se acerca a 8/3 = 2.667 # Ya con datos o una funcion, en una linea (sin bucle): # trapezoid(y, x) | simpson(y, x=x) | quad(func, a, b)

Del límite a la práctica: ¿cuál uso?

Con muchos rectángulos la suma se acerca a 8/3. Ya con datos: trapezoid o simpson. Con una fórmula y precisión máxima: quad (devuelve valor y error).

El teorema fundamental, en Python

La derivada del área acumulada devuelve la función; y el atajo F(b) − F(a)

import numpy as np from scipy.integrate import quad, cumulative_trapezoid # Parte 2: quad debe coincidir con F(b) - F(a) f = lambda x: np.sin(x) val, err = quad(f, 0, np.pi) # ~ 2.0 # F(x) = -cos(x) -> F(pi) - F(0) = 1 + 1 = 2 # Parte 1: derivar el area acumulada devuelve f x = np.linspace(0, np.pi, 1000) A = cumulative_trapezoid(np.sin(x), x, initial=0) dA = np.gradient(A, x) # ~ sin(x) = f(x)

Las dos partes, comprobadas con código

Parte 2: quad da lo mismo que evaluar la antiderivada en los extremos. Parte 1: derivar la curva acumulada (cumulative_trapezoid) reconstruye la función original. Acumular y derivar son inversas.

El puente estrella: teorema fundamental y probabilidad

La CDF es la integral de la densidad; el muestreo inverso la usa al revés

De la densidad a las muestras

La CDF cumple F(x)=∫ p(t) dt, así que por el teorema fundamental p(x) = F'(x). El muestreo por transformada inversa genera datos con X = F⁻¹(U), donde U es uniforme(0,1).

import numpy as np rng = np.random.default_rng(0) # F es la integral de la densidad p (CDF de una Beta(2,2)) x = np.linspace(0, 1, 1000) F = 3*x**2 - 2*x**3 # Muestreo por transformada inversa: X = F^{-1}(U) U = rng.random(10000) muestras = np.interp(U, F, x) # invierte la CDF

Es la base de la simulación Monte Carlo y de varios modelos generativos: acumular una densidad y luego invertir.

Sustitución en SymPy

La regla de la cadena al revés, verificada por el computador

import sympy as sp x = sp.symbols('x') # u = x**2 -> du = 2x dx sp.integrate(2*x*sp.exp(x**2), x) # -> exp(x**2) sp.integrate(x*sp.cos(x**2 + 1), x) # -> sin(x**2 + 1)/2 # En definidas, cambiar tambien los limites: sp.integrate(2*x*sp.exp(x**2), (x, 0, 1)) # -> E - 1

El criterio

Buscar una función interna u=g(x) cuya derivada aparezca (salvo constante) como factor. Aquí u=x²: su derivada 2x está en el integrando.

En definidas, ojo con los límites

Al cambiar de variable hay que cambiar también los límites, o volver a x al final. Nunca las dos cosas: es el error más común del tema.

Área entre curvas en Python

Integrar la diferencia superior − inferior

El mismo ejemplo del pizarrón

La región entre √x y x² en [0,1] tiene área 1/3. En Python es integrar la resta de las dos funciones, y fill_between la dibuja.

import numpy as np from scipy.integrate import quad import matplotlib.pyplot as plt f = lambda x: np.sqrt(x) g = lambda x: x**2 area, _ = quad(lambda x: f(x) - g(x), 0, 1) # -> 1/3 x = np.linspace(0, 1, 200) plt.fill_between(x, f(x), g(x), where=(f(x) > g(x)), interpolate=True, alpha=0.3)

interpolate=True calcula el cruce exacto y evita triángulos sin rellenar. Esta idea de "área bajo una curva" es la puerta al AUC.

El AUC-ROC: una integral que ya usan sin saberlo

El área bajo la curva ROC se calcula con la regla del trapecio

La conexión con ciencia de datos

Cuando evalúan un clasificador con AUC, sklearn.metrics.auc(fpr, tpr) calcula el área bajo la ROC literalmente con la regla del trapecio.

from sklearn.metrics import roc_curve, auc import numpy as np fpr, tpr, thr = roc_curve(y_true, y_score) roc_auc = auc(fpr, tpr) # area bajo la ROC # es EXACTAMENTE lo mismo que: np.trapezoid(tpr, fpr) # regla del trapecio

AUC = 1.0 es perfecto; 0.5 es azar. Es la probabilidad de ordenar un positivo por encima de un negativo al azar.

El área bajo la curva, con datos del país

Demanda eléctrica del SEN (CENCE / ICE): potencia → energía

El concepto físico perfecto

El CENCE publica la demanda como potencia en MW. Al integrar potencia (MW) en el tiempo (h) se obtiene energía (MWh): el área bajo la curva es la misma cantidad de su recibo de electricidad.

from scipy.integrate import trapezoid, cumulative_trapezoid # h = horas (0..23), d = demanda en MW (curva del CENCE / ICE) energia_dia = trapezoid(d, h) # MWh del dia (area total) acumulada = cumulative_trapezoid(d, h, initial=0) # curva acumulada

trapezoid da la energía total; cumulative_trapezoid da la curva de energía acumulada a lo largo del día.

Volúmenes de revolución (panorama)

Un vistazo: no es el foco del curso, pero cierra el tema de integrales

Panorama, no evaluado a mano

Girar una curva para llenar un volumen

Al girar una curva alrededor de un eje se genera un sólido. El volumen también es una integral, según el método:

  • Discos: V = π∫ [R(x)]² dx
  • Arandelas: V = π∫ ([R]² − [r]²) dx
  • Cascarones: V = 2π∫ r(x)·h(x) dx

En ciencia de datos su uso es de nicho: generación de mallas 3D y CAD (por ejemplo con PyVista). Lo dejamos como cultura general.

¿Por qué integrar sigue importando en deep learning?

No es matemática de museo: está en el corazón de los modelos

Valor esperado

El valor esperado de una variable continua es una integral: E[g(X)] = ∫ g(x) f(x) dx. Las pérdidas esperadas son integrales.

Probabilidad

Toda densidad debe integrar 1 (normalización). La CDF es la integral de la PDF. Sin integrales no hay distribuciones bien definidas.

Modelos generativos

Difusión, flujos normalizadores y ODE neuronales se basan en integrar ecuaciones diferenciales.

Métodos como Monte Carlo nacen de reescribir una integral como un valor esperado y estimarlo con un promedio muestral: E[g(X)] ≈ (1/n) Σ g(Xᵢ). La conexión más profunda entre integración y ciencia de datos

Lo que se llevan de hoy

Tres ideas para no olvidar

1. El teorema fundamental es el atajo

Para una integral definida —que en principio suma infinitos puntos— basta hallar una antiderivada F y evaluarla en dos: ∫ₐᵇ f = F(b) − F(a). Acumular y derivar son inversas.

2. El área bajo la curva está en todas partes

Energía = área bajo la potencia, AUC = área bajo la ROC, probabilidad = área bajo la densidad. Una idea geométrica que un científico de datos calcula a diario.

3. Usen las APIs de 2026

np.trapezoid, scipy.integrate.{trapezoid, simpson, quad, cumulative_trapezoid}. Nunca np.trapz.
