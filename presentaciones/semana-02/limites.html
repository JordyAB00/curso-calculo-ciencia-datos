SymPy: límites en una línea

Versión 1.14.0 (abril 2025) — la que tienen instalada

from sympy import symbols, limit, oo, sin, cos, exp x = symbols('x', real=True) # Límite básico limit(sin(x)/x, x, 0) # Resultado: 1 # El ejemplo 0/0 de la parte manual limit((x**2 - 4)/(x - 2), x, 2) # Resultado: 4 # Límite al infinito (oo = doble letra o minúscula) limit((1 + 1/x)**x, x, oo) # E (la constante de Euler) # Límite de la sigmoide cuando x → −∞ limit(1/(1 + exp(-x)), x, -oo) # Resultado: 0

Por qué SymPy puede esto

Internamente usa heurísticas para casos comunes y, cuando fallan, recurre al algoritmo de Gruntz (un método sofisticado para límites de funciones reales).

Para qué nos sirve

Para verificar todo lo que calculen a mano. SymPy es nuestro juez imparcial: ya no tienen que confiar en el profesor ni en sus apuntes.

Límites laterales en SymPy

El tercer argumento de limit() define la dirección

from sympy import symbols, limit, oo x = symbols('x') # Límite bilateral (no existe en el caso de 1/x) limit(1/x, x, 0) # zoo (complejo infinito, "no dirigido") # Por la derecha: x → 0+ limit(1/x, x, 0, '+') # oo # Por la izquierda: x → 0− limit(1/x, x, 0, '-') # -oo # Ambos lados explícitos limit(1/x, x, 0, '+-') # zoo

El resultado zoo en SymPy significa "complejo infinito sin dirección". Es la forma que tiene SymPy de decirles que el límite no existe en sentido finito porque los laterales van por caminos opuestos.

Advertencia: limit() vs subs()

El error más común al usar SymPy para evaluar comportamiento asintótico

# Estilo incorrecto expr = sin(x)/x expr.subs(x, 0) # Output: nan (forma 0/0) expr.subs(x, oo) # Output: nan también # Razón: subs() reemplaza el # símbolo y evalúa directamente, # sin entender comportamiento # asintótico ni indeterminadas.
# Estilo correcto expr = sin(x)/x limit(expr, x, 0) # Output: 1 (correcto) limit(expr, x, oo) # Output: 0 (correcto) # limit() aplica algoritmos # que reconocen las formas # 0/0, ∞/∞, 0·∞, 1^∞, etc.

La documentación oficial de SymPy lo advierte explícitamente: "incluso aunque SymPy tiene objetos para representar infinito, usarlos para evaluación no es confiable porque no rastrean cosas como tasa de crecimiento". Regla: siempre limit(), nunca subs() con infinitos.

Continuidad: por qué importa en ciencia de datos

Recordatorio aplicado de las tres condiciones

1
f(a) está definida
2
El límite cuando x → a existe
3
Ese límite vale f(a)

El detalle sutil que importa para deep learning

Una función puede ser continua sin ser diferenciable. Continua significa "se dibuja sin levantar el lápiz". Diferenciable significa además que "no tiene esquinas puntiagudas". ReLU es continua en todo R, pero tiene una esquina en x = 0 donde no es diferenciable.

Los optimizadores como SGD, Adam o Muon necesitan calcular gradientes. Si la función de pérdida tiene saltos o esquinas problemáticas, el entrenamiento se rompe. Por eso la elección de activación es una decisión sobre límites y continuidad.

Discontinuidades clasificadas con SymPy

Diagnóstico automático en Colab

from sympy import symbols, limit, oo, Piecewise, sin x = symbols('x') # Evitable: (x²−1)/(x−1) en x=1 f1 = (x**2 - 1)/(x - 1) limit(f1, x, 1, '-'), limit(f1, x, 1, '+') # (2, 2) → laterales coinciden pero f(1) no existe → EVITABLE # De salto: Heaviside en x=0 f2 = Piecewise((0, x < 0), (1, True)) limit(f2, x, 0, '-'), limit(f2, x, 0, '+') # (0, 1) → laterales distintos → SALTO # Infinita: 1/x² en x=0 f3 = 1/x**2 limit(f3, x, 0) # oo → límite infinito → ASINTÓTICA

Patrón mental: si los laterales coinciden pero la función no, es evitable. Si los laterales difieren, es salto. Si al menos uno es infinito, es asintótica.

ReLU: el caso más famoso de continua sin ser diferenciable

La activación más usada en deep learning durante una década

Análisis matemático

ReLU(x) = max(0, x). Es continua en todo R (los laterales en cualquier punto coinciden con el valor de la función). Pero en x = 0:

  • Derivada por la izquierda: 0
  • Derivada por la derecha: 1
  • Como no coinciden, la derivada NO existe en 0
import torch x = torch.tensor(0.0, requires_grad=True) y = torch.relu(x) y.backward() print(x.grad.item()) # Output: 0.0 # PyTorch elige 0 por convención. # JAX elige 0. TensorFlow elige 0. # Es una decisión de ingeniería, # no de matemática.

Los frameworks resuelven la ambigüedad declarando que la derivada en 0 vale 0. Esto se llama subgradiente y, aunque matemáticamente no es la derivada estricta, basta para que el descenso por gradiente converja. Es el ejemplo más claro de cómo la teoría se adapta a la práctica.

Por qué nacieron GELU, SiLU y Mish

Activaciones suaves: diferenciables infinitas veces

La motivación

ReLU funciona bien pero tiene la "esquina" en 0. Las activaciones modernas suavizan esa esquina manteniendo el comportamiento general: cero para entradas muy negativas, lineal para entradas muy positivas.

GELU

Gaussian Error Linear Unit. Es la activación estándar en BERT, GPT-2 y ViT. Su forma es continua y suave en todo R, con lim x→-∞ igual a 0 y asíntota oblicua y=x cuando x→+∞.

SiLU (Swish)

SiLU(x) = x · σ(x). Suave en todo R. Es la activación elegida por Llama, PaLM y muchos modelos open source recientes. Mismos límites que GELU.

Mish

Mish(x) = x · tanh(ln(1+e^x)). Suave, no monótona en la zona negativa cercana a 0. Resultados ligeramente mejores en algunos benchmarks de visión.

Las tres comparten el mismo comportamiento asintótico: se "apagan" hacia 0 cuando la entrada es muy negativa, y se vuelven lineales cuando la entrada es muy positiva. La diferencia es la suavidad de la transición.

Sigmoide y tanh son las únicas acotadas en ambos extremos. Esto las hace útiles para salidas de probabilidades pero peligrosas en capas profundas, como veremos ahora.

Vanishing gradient: el problema clásico

Cuando los límites laterales producen gradientes nulos

Cuando la sigmoide recibe una entrada muy positiva (digamos 10), su salida está prácticamente en su asíntota horizontal 1. La derivada de la sigmoide en esa zona es casi cero. En una red profunda, multiplicar muchos "casi cero" da exactamente cero.

import torch # Entradas en regiones de saturación de la sigmoide x = torch.tensor([-100.0, -10.0, 0.0, 10.0, 100.0], requires_grad=True) y = torch.sigmoid(x).sum() y.backward() print(x.grad) # tensor([3.7835e-44, 4.5396e-05, 2.5000e-01, # 4.5396e-05, 3.7835e-44]) # En x=±100 el gradiente es prácticamente cero. # En 5 capas: 4.5e-5 elevado a la 5 ≈ 1.9e-22 # Las primeras capas dejan de aprender por completo.

Esto explica por qué desde 2012 (AlexNet) las capas ocultas dejaron de usar sigmoide. ReLU y sus variantes mantienen un gradiente constante en el lado positivo, evitando el desvanecimiento.

Estabilidad numérica: límites mal calculados rompen modelos

La industria nunca implementa log(sigmoid(x)) directamente

# Implementación ingenua (peligrosa) import numpy as np x = -1000 sigmoid_x = 1/(1 + np.exp(-x)) # sigmoid(-1000) = 0.0 exacto # por subdesbordamiento np.log(sigmoid_x) # log(0) = -inf # Si esto entra a una pérdida, # el entrenamiento se rompe # con NaN inmediatamente.
# Implementación profesional from scipy.special import log_expit x = -1000 log_expit(x) # Output: -1000.0 (correcto) # Internamente NO calcula # log(1/(1+exp(-x))) directo. # Usa una reformulación que # preserva precisión sin # pasar por el infinito.

Funciones que ya tienen estabilidad asegurada

  • scipy.special.expit: sigmoide estable para valores extremos
  • scipy.special.log_expit: log de la sigmoide sin pasar por log(0)
  • scipy.special.logsumexp: trucos para sumar exponenciales grandes
  • torch.nn.BCEWithLogitsLoss: fusión segura de sigmoide y log-loss
  • torch.nn.CrossEntropyLoss: fusión segura de softmax y log-loss

Una serie real con comportamiento asintótico

Contexto

import numpy as np from scipy.optimize import curve_fit # Modelo de decaimiento asintótico def modelo(t, a, b, k): return a + b * np.exp(-k * t) # El parámetro a representa la asíntota horizontal: # el equilibrio de largo plazo al que la TBP converge. # Si ajustamos a la serie, recuperamos lim t→∞ TBP(t).

Este es un caso de límite al infinito en datos reales: la TBP no oscila, decae monótonamente hacia un nuevo equilibrio. Calcular ese límite es predecir hacia qué valor se está estabilizando.

Caso costarricense: proyecciones poblacionales del INEC

La curva logística en documentos oficiales de 2025

Cita literal del documento metodológico del INEC (junio 2025)

"La función logística cuenta con valores teóricos del máximo y mínimo del crecimiento poblacional; es decir, con dicha función se plantea el supuesto de que con un valor máximo (cota superior) y un valor mínimo (cota inferior) del patrón de crecimiento."

import numpy as np import matplotlib.pyplot as plt # Modelo logístico estándar (INEC usa este) def logistica(t, L, k, t0): return L / (1 + np.exp(-k * (t - t0))) # Asíntotas (los límites al infinito): # lim t→+∞ logistica(t) = L (capacidad de carga) # lim t→-∞ logistica(t) = 0 (origen poblacional) # El INEC proyecta hacia 2050 con esta estructura. # L es el límite poblacional al que tiende el cantón.

El INEC, en documentos oficiales publicados en junio de 2025, calcula explícitamente los límites al infinito de funciones logísticas para proyectar la población de cada cantón hasta 2050. La matemática que vimos hoy se usa en las decisiones de política pública costarricenses.

Cuando SymPy no llega a una respuesta

Casos límite del solucionador y cómo manejarlos

Situaciones reportadas en el tracker oficial de SymPy

  • Algunos límites con valor absoluto quedan sin evaluar y devuelven un objeto Limit en lugar de un valor
  • Combinaciones complejas con exponenciales y raíces pueden necesitar reescritura previa
  • L'Hôpital aplicada internamente a veces produce resultados parciales
  • Mezclar numpy.inf con sympy.oo en la misma expresión causa errores silenciosos
from sympy import limit, oo from mpmath import mp, mpf # Buena práctica: verificar SymPy con mpmath numérico mp.dps = 50 # 50 dígitos decimales de precisión resultado_sympy = limit(sin(x)/x, x, 0) resultado_mpmath = mp_limit(lambda x: sin(x)/x, 0) # Ambos deben coincidir hasta el nivel de precisión

Regla de oro: si SymPy entrega un valor cerrado, verificarlo numéricamente con mpmath a alta precisión. Si SymPy entrega un objeto Limit sin evaluar, intentar reescribir la expresión o cambiar de variable.

El equipo de Moonshot AI observó que durante el entrenamiento "los máximos de los logits de atención superan rápidamente magnitudes de 1000". Propusieron QK-LN combinado con softmax capping para permitir tasas de aprendizaje 1.5x mayores sin divergencia. Toda la maquinaria se basa en analizar comportamiento límite de las activaciones internas.

El research más reciente en deep learning es, en el fondo, control sofisticado de límites al infinito. Lo que aprendieron hoy es el vocabulario base de papers publicados en NeurIPS y arXiv hace seis meses.

Errores comunes en estudiantes de ciencia de datos

Lo que suele aparecer en talleres y parciales

Confundir f(a) con el límite

El límite de (x²−1)/(x−1) cuando x → 1 es 2, aunque la función no exista en x = 1. Son cosas distintas.

Usar subs() con infinito

expr.subs(x, oo) devuelve nan en formas indeterminadas. Siempre usar limit() para evaluar comportamiento asintótico.

Creer que continuo implica diferenciable

ReLU es el contraejemplo central. Es continua en R pero no diferenciable en 0. Las dos propiedades son distintas.

Mezclar numpy.inf con sympy.oo

Son objetos diferentes. SymPy trabaja con su propio oo. NumPy con np.inf. Confundirlos produce errores raros y silenciosos.

Calcular log(0) sin estabilizar

Implementar log(sigmoid(x)) directamente colapsa en valores extremos. Usar siempre log_expit, logsumexp, BCEWithLogitsLoss.

Olvidar verificar resultados

SymPy a veces entrega objetos Limit sin evaluar. La buena práctica es comparar con evaluación numérica (mpmath o NumPy con valores cercanos al punto).
